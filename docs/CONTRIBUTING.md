# 🤝 Contributing Guide\n\nThank you for your interest in contributing to Naar & Noor! This guide will help you get started.\n\n## Code of Conduct\n\nPlease be respectful and professional in all interactions. We are committed to providing a welcoming and inclusive environment.\n\n## Getting Started\n\n1. Fork the repository\n2. Clone your fork\n3. Create a feature branch\n4. Make your changes\n5. Submit a pull request\n\n## Development Setup\n\nSee [Getting Started](./GETTING_STARTED.md) for detailed setup instructions.\n\n## Branching Strategy\n\n### Branch Naming\n\n- `feature/description` - New features\n- `bugfix/description` - Bug fixes\n- `hotfix/description` - Production hotfixes\n- `docs/description` - Documentation updates\n- `refactor/description` - Code refactoring\n\n### Example\n\n```bash\ngit checkout -b feature/add-user-authentication\n```\n\n## Commit Messages\n\nUse clear, descriptive commit messages:\n\n```\n[TYPE] Brief description\n\nDetailed explanation of changes (optional)\n\nFixes #123\n```\n\n### Types\n\n- `feat:` New feature\n- `fix:` Bug fix\n- `docs:` Documentation\n- `style:` Code style changes\n- `refactor:` Code refactoring\n- `test:` Test additions/changes\n- `chore:` Build/dependency changes\n\n### Examples\n\n```\nfeat: Add reservation confirmation email\n\nImplement email notification when reservation is confirmed.\nUses SendGrid for email delivery.\n\nFixes #456\n```\n\n```\nfix: Resolve menu item price display issue\n\nFix decimal formatting for menu item prices in the UI.\n```\n\n## Code Style\n\n### Backend (C#)\n\n- Follow [Microsoft C# Coding Conventions](https://docs.microsoft.com/dotnet/csharp/fundamentals/coding-style/coding-conventions)\n- Use PascalCase for class names and methods\n- Use camelCase for variables and parameters\n- Use meaningful names\n- Keep methods focused and small\n- Add XML documentation comments\n\n**Example:**\n\n```csharp\n/// <summary>\n/// Gets all available menu items.\n/// </summary>\n/// <returns>List of menu items</returns>\npublic async Task<List<MenuItem>> GetMenuItemsAsync()\n{\n    return await _context.MenuItems\n        .Where(m => m.IsAvailable)\n        .ToListAsync();\n}\n```\n\n### Frontend (TypeScript/Angular)\n\n- Follow [Angular Style Guide](https://angular.io/guide/styleguide)\n- Use camelCase for variables and functions\n- Use PascalCase for classes and components\n- Use meaningful names\n- Keep components focused\n- Add JSDoc comments for public methods\n\n**Example:**\n\n```typescript\n/**\n * Fetches menu items from the API\n * @returns Observable of menu items\n */\ngetMenuItems(): Observable<MenuItem[]> {\n  return this.http.get<MenuItem[]>(`${this.apiUrl}/menu`);\n}\n```\n\n## Testing\n\n### Backend Tests\n\nWrite unit tests for business logic:\n\n```bash\ncd api-server\ndotnet test\n```\n\n**Example Test:**\n\n```csharp\n[Fact]\npublic async Task CreateReservation_WithValidData_ReturnsReservationId()\n{\n    // Arrange\n    var command = new CreateReservationCommand\n    {\n        GuestName = \"John Doe\",\n        Email = \"john@example.com\",\n        PhoneNumber = \"+1-555-0123\",\n        ReservationDate = DateTime.Now.AddDays(7),\n        NumberOfGuests = 4\n    };\n\n    // Act\n    var result = await _handler.Handle(command, CancellationToken.None);\n\n    // Assert\n    Assert.True(result > 0);\n}\n```\n\n### Frontend Tests\n\nWrite unit tests for components and services:\n\n```bash\ncd naar-noor\nnpm test\n```\n\n**Example Test:**\n\n```typescript\ndescribe('MenuComponent', () => {\n  let component: MenuComponent;\n  let fixture: ComponentFixture<MenuComponent>;\n\n  beforeEach(async () => {\n    await TestBed.configureTestingModule({\n      imports: [MenuComponent]\n    }).compileComponents();\n\n    fixture = TestBed.createComponent(MenuComponent);\n    component = fixture.componentInstance;\n  });\n\n  it('should display menu items', () => {\n    component.items = [\n      { id: 1, name: 'Tandoori Chicken', price: 14.99 }\n    ];\n    fixture.detectChanges();\n\n    const compiled = fixture.nativeElement;\n    expect(compiled.querySelector('h2').textContent).toContain('Tandoori Chicken');\n  });\n});\n```\n\n## Pull Request Process\n\n1. Update documentation if needed\n2. Add/update tests\n3. Ensure all tests pass\n4. Create pull request with clear description\n5. Link related issues\n6. Request review from maintainers\n7. Address review comments\n8. Merge when approved\n\n### PR Template\n\n```markdown\n## Description\nBrief description of changes\n\n## Type of Change\n- [ ] Bug fix\n- [ ] New feature\n- [ ] Breaking change\n- [ ] Documentation update\n\n## Related Issues\nFixes #123\n\n## Testing\nDescribe testing performed\n\n## Checklist\n- [ ] Code follows style guidelines\n- [ ] Tests added/updated\n- [ ] Documentation updated\n- [ ] No new warnings generated\n```\n\n## Documentation\n\n- Update README if adding new features\n- Add comments for complex logic\n- Update API documentation for endpoint changes\n- Keep docs in sync with code\n\n## Performance Considerations\n\n- Avoid N+1 queries\n- Use pagination for large datasets\n- Implement caching where appropriate\n- Optimize database queries\n- Minimize bundle size\n\n## Security\n\n- Validate all inputs\n- Use parameterized queries\n- Never commit secrets\n- Follow OWASP guidelines\n- Use HTTPS in production\n- Implement proper authentication/authorization\n\n## Common Issues\n\n### Port Already in Use\n\n```bash\n# Frontend\nnpm run dev -- --port 4300\n\n# Backend\ndotnet run --project src/NaarNoor.API/NaarNoor.API.csproj --urls \"http://localhost:5001\"\n```\n\n### Database Connection Issues\n\n1. Verify SQL Server is running\n2. Check connection string\n3. Ensure database exists\n4. Check user permissions\n\n### Node Modules Issues\n\n```bash\nrm -rf node_modules package-lock.json\nnpm install\n```\n\n## Getting Help\n\n- Check existing issues and discussions\n- Review documentation\n- Ask in pull request comments\n- Create a new issue with detailed information\n\n## Recognition\n\nContributors will be recognized in:\n- CONTRIBUTORS.md file\n- Release notes\n- Project README\n\n## License\n\nBy contributing, you agree that your contributions will be licensed under the same license as the project.\n\n## Questions?\n\nFeel free to open an issue or discussion if you have questions about contributing.\n\nThank you for contributing to Naar & Noor!\n
